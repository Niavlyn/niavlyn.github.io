ETSI Augmented Reality Framework

Implementation of an augmented reality standard using Unity and ARFoundation for Android and iOS.

Context

During the last internship of my Master's degree, I worked at Orange Innovation, on a Augmented Reality (AR) project. I worked on an implementation of a new standard of the European Telecommunications Standards Institute (ETSI), of which Orange is a partner. The objectives of the new standard are summarized by ETSI as follows :

"We are defining a framework for the interoperability of Augmented Reality (AR) components, systems and services that specifies relevant components and interfaces required for an AR solution. Augmented Reality is the ability to mix in real-time spatially-registered digital content with the real world surrounding the user.
The development of a modular architecture will allow components from different providers to interoperate through defined interfaces. Transparent and reliable interworking between different AR components is key to the successful roll-out and wide adoption of AR applications and services."

Source here : ETSI ARF

My work

In this project, I worked on the implementation of a component named World Analysis, which task was to relocalize content in the AR scene at the correct places. I made this implementation using Unity and the ARFoundation framework, using the specifications of the APIs created by my colleagues.
At the end of my internship, I created a use case demo for Android and iOS devices that illustrates how the ARF framework works.
The project is meant to be open source, you can find the GitLab repository below (my part is in the World Analysis Wrapper / Validation folders).

Using a combination of real life "trackable" objects and fixed space positions known as anchors, we are able to deliver an extremely similar experience to the user regardless of the device used.

Supercopter

VR game that uses MYO armbands to control an aircraft.

Context

Supercopter is a university team project I worked on with other students during my master’s degree as an end-of-year project. The goal was to apply our class learnings in a single, collaborative project. We decided to create a VR game for Oculus Quest 1 and 2 headsets.
In this game, the player controls a "supercopter" and is tasked with taking down a giant eagle flying between skyscrapers. The eagle regularly drops feathers in the air. Players can gain points by collecting feathers or by hitting the eagle with the supercopter, which makes the eagle reappear in another location on the map. The eagle's flight path was created using reinforcement learning. There are two ways to control the supercopter: the classic way, using a keyboard, or the fun wayusing a MYO armband that detects forearm movements to move the supercopter. (The armbands are not very reliable, so playing with the keyboard is much easier.)

We used the MYO armband to control the supercopter, but it was difficult to use consistently, so we kept the keyboard option, which was originally intended only for debugging.

AVATAR

Experiment on the human interactions with virtual avatars with realistic expressions in VR.

Context

AVATAR was a team project I started during the first year of my master’s degree and later continued as an internship for the IRISA and LARIS university laboratories. The aim of the project was to implement realistic facial behavior in virtual avatars, covering both movement and facial expressions. The entire project was designed to support a PhD student’s thesis on the use of realistic virtual avatars in the context of virtual reality learning.
We used the university’s motion capture room to record around 20 different 3D poses for our avatars, capturing both body and facial data with a set of markers on the actor. We also used special glasses to record eye movements. We then spent significant time cleaning the motion capture (MOCAP) data before merging it with the eye-tracking data to achieve a fully accurate and realistic human behavior model.

This is the marker set (the disposition of markers on the actor's body) we used for our MOCAP sessions. We also added markers on the actor's face.

This is the eye-tracking device we used to record the actor’s eye movements. It caused some issues because its plastic parts obstructed the facial markers, making them difficult for the MOCAP cameras to detect and degrading the quality of our facial MOCAP data.

My work

During the team project phase, I worked on processing the MOCAP data for the body, cleaning it and importing it onto our avatar in the Unity game engine. I also implemented the map and VR functionalities. In the second phase, I worked for one month as an intern to complete the project. I replaced the avatar with one that had blendshapes already integrated for the face (blendshapes are predefined variations of a 3D model’s mesh that enable smooth transitions between different facial expressions, shapes, or poses by interpolating vertex positions). I also used new facial MOCAP data, as the data we initially recorded was too low-quality for realistic expressions.
