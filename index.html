ETSI Augmented Reality Framework

Implementation of an augmented reality standard using Unity and ARFoundation for Android and iOS.

Context

During the last internship of my Master's degree, I worked at Orange Innovation, on a Augmented Reality (AR) project. I worked on an implementation of a new standard of the European Telecommunications Standards Institute (ETSI), of which Orange is a partner. The objectives of the new standard are summarized by ETSI as follows :

"We are defining a framework for the interoperability of Augmented Reality (AR) components, systems and services that specifies relevant components and interfaces required for an AR solution. Augmented Reality is the ability to mix in real-time spatially-registered digital content with the real world surrounding the user.
The development of a modular architecture will allow components from different providers to interoperate through defined interfaces. Transparent and reliable interworking between different AR components is key to the successful roll-out and wide adoption of AR applications and services."

Source here : ETSI ARF

My work

In this project, I worked on the implementation of a component named World Analysis, which task was to relocalize content in the AR scene at the correct places. I made this implementation using Unity and the ARFoundation framework, using the specifications of the APIs created by my colleagues.
At the end of my internship, I created a use case demo for Android and iOS devices that illustrates how the ARF framework works.
The project is meant to be open source, you can find the GitLab repository below (my part is in the World Analysis Wrapper / Validation folders).

Using a combination of real life "trackable" objects and fixed space positions known as anchors, we are able to deliver an extremely similar experience to the user regardless of the device used.
